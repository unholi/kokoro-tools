Quick Start · README

Kokoro
Audiobook Studio

A local, offline, browser-based tool for converting text files into audiobook-quality narration using the Kokoro TTS engine.

What's in this package
kokoro-tools/
kokoro-studio required Shell launcher — run this to start the app
serve.py required Local web server (port 7891)
kokoro_run.py required TTS worker — converts text to WAV
kokoro_ui.html required Browser interface
voices.txt required List of available Kokoro voice files
voice_sampler.py optional Generates native-language voice previews
voice_sampler_accented_english.py optional Generates accented-English voice previews
user_guide.html optional Full in-app user guide (opens from the ? Help button)
README.html optional This file
Getting started
1
Prep Environment
You will run the Kokoro as a non-root user. Create the following directories in the user's home directory. 
mkdir -p ~/Downloads/kokoro    # (location for the bulk of Kokoro's software)
mkdir -p ~/.local/bin/         # (location for the Kokoro Audiobook Studio app)
2
Install Kokoro dependencies
You need Python 3.10+, Python PIP, espeak-ng, python3.12-venv and create a virtual environment with Kokoro installed. The Kokoro studio expects the Python virtual environment (venv) to exist at ~/Downloads/kokoro/env1/. 
# Patch & Install OS Dependencies
sudo apt update ; sudo apt upgrade -y
apt install -y python3 python3-pip pipx espeak-ng python3.12-venv

# Create Python Virtual Environment
python3 -m venv env1

# Enter Python Virtual Environment
source env1/bin/activate

# Download Kokoro Python Dependencies
# Audio I/O Support
pip install soundfile

# PyTorch — CPU-only build (smaller download, sufficient for most systems)
pip install torch --index-url https://download.pytorch.org/whl/cpu

# Download Core Kokoro A.I. Model and Code (approx. 5.7 GB)
pip install kokoro
See user_guide.html → Section 3 for the full install walkthrough including GPU options and pre-caching all voice files.
3
Place the kokoro-tools folder
Copy the kokoro-tools.tar.gz file into ~/.local/bin/kokoro-tools/ Unzip the file: 
cp /path/to/kokoro-tools.tar.gz  ~/.local/bin/
cd ~/.local/bin/
tar xvzf ./kokoro-tools.tar.gz
./kokoro-tools/
./kokoro-tools/serve.py
./kokoro-tools/voices.txt
./kokoro-tools/kokoro_ui.html
./kokoro-tools/kokoro-studio
./kokoro-tools/README.html
./kokoro-tools/user_guide.html
./kokoro-tools/kokoro_run.py
./kokoro-tools/voice_sampler.py
./kokoro-tools/voice_sampler_accented_english.py
This location is already on $PATH for user-installed binaries on Ubuntu 24.04 / Pop!_OS. If you use a different path, update VENV_ACTIVATE inside the kokoro-studio script to match your venv location.
4
Make the launcher executable
chmod +x ~/.local/bin/kokoro-tools/kokoro-studio
You only need to do this once.
5
Add a shell alias (recommended)
So you can type kokoro-studio from anywhere: 
echo "alias kokoro-studio='~/.local/bin/kokoro-tools/kokoro-studio'" >> ~/.bashrc
source ~/.bashrc
6
Run the studio from your project folder
Navigate to the directory containing your .txt chapter files, then launch: 
cd ~/my-audiobook-project
kokoro-studio
The server starts, your browser opens to http://localhost:7891, and the interface is ready. The terminal window that launched the studio must stay open while you work.
7
Generate voice samples (first time only)
The Voice Audition panel needs pre-generated sample WAV files to play previews. Inside Kokoro Audiobook Studioweba app, open the Voice Audition section and click ⚙ Native language and/or ⚙ Accented English to generate them.
Run one voice generator at a time.
Depending on your system, this can takes several minutes and only needs to be done once — samples are saved inside kokoro-tools/ and reused on every future launch.

Platform Notes
⚠

Ubuntu / Pop!_OS 24.04: These instructions are written for Ubuntu 24.04 and Pop!_OS 24.04. Commands and paths should work as-is on most Debian-based distributions. macOS and Windows users will need to adjust the venv activation path, the apt commands, and the terminal emulator detection inside kokoro-studio.

ℹ

Fully offline after setup: Kokoro downloads model weights (~330 MB) and voice files from Hugging Face on first use. After that, no internet connection is required. To lock it offline permanently, add export HF_HUB_OFFLINE=1 to your ~/.bashrc.

EXTRA: Preparing your text files
8
Format your .txt files for best results
Kokoro reads plain text — no markup, no formatting codes. How you punctuate and structure the text directly controls pacing, pausing, and intonation. Follow these rules before running a conversion.

Punctuation = pacing
Kokoro uses punctuation to determine pauses, not blank lines.
  • Comma , — brief pause
  • Period . — full stop, natural breath
  • Em dash — — dramatic mid-sentence break
  • Ellipsis ... — trailing off, hesitation

Spell out numbers and abbreviations
The phonemiser handles most cases but stumbles on edge cases. 
Use:  "Chapter Three"   not  "Chapter 3"
Use:  "Doctor Voss"     not  "Dr. Voss"
Use:  "twenty-five"     not  "25"
Use:  "and"             not  "&"

Fix curly quotes before converting
Word processors save “smart” quotes that can confuse the phonemiser. Run this from your chapter directory to convert them all at once: 
sed -i "s/‘/'/g; s/’/'/g; s/“/"/g; s/”/"/g" *.txt

Proper nouns and invented names
Kokoro mispronounces unusual or invented names. Write them phonetically in the source file — e.g. if Kaelith should sound like Kaylith, use the phonetic spelling. Keep a substitution log so you apply changes consistently across all chapters.

Sentence length sweet spot
Sentences over ~40 words can lose natural intonation mid-way — break them with a period or em dash. Very short sentences (under 5 words) can sound slightly clipped; fine for action beats, avoid in exposition and emotional passages.

File encoding
Save every file as UTF-8. The studio falls back to latin-1 if needed, but UTF-8 is safer and avoids character rendering surprises.

One chapter per file, headers optional
Keep each chapter as a single .txt file. Splitting chapters resets Kokoro’s prosody (rhythm and intonation) at each boundary, which can make narration sound disjointed at join points. If your file starts with a chapter heading like “Chapter One: The Storm”, Kokoro will read it aloud — this is usually desirable; delete it only if you’re adding chapter markers separately in post-production.

Kokoro Optimization Prompt
When preparing a manuscript for Kokoro, use the prompt below. It is designed to handle the "phonetic translation" and structural cleanup necessary to prevent robotic pacing or mispronunciations. 
# AI Prompt for Kokoro TTS Optimization
Act as a specialized script editor for the Kokoro Text-to-Speech engine. Your goal is to rewrite the provided text into a "clean" format that maximizes natural intonation and pacing.

Follow these strict transformation rules:

1. **Pacing & Punctuation:**
   - Use commas (,) for brief pauses.
   - Use periods (.) for full stops and breaths.
   - Use em dashes (—) for dramatic mid-sentence breaks.
   - Use ellipses (...) for trailing off or hesitations.
   - Ensure no sentence exceeds ~40 words; break long sentences using periods or em dashes to maintain prosody.
   - Expand very short sentences (under 5 words) in exposition to avoid a "clipped" sound, unless they are specific action beats.

2. **Verbal Expansion:**
   - Spell out all numbers (e.g., "twenty-five" instead of "25", "Chapter Three" instead of "Chapter 3").
   - Expand all abbreviations and symbols (e.g., "Doctor" instead of "Dr.", "and" instead of "&").

3. **Phonetic Formatting:**
   - If the text contains invented names or unusual proper nouns, provide a phonetic spelling that is easy for a phonemiser to interpret (e.g., change "Kaelith" to "Kaylith").

4. **Formatting Stripping:**
   - Remove all Markdown (bold, italics, headers) and special formatting codes.
   - Convert all "smart" or curly quotes (“ ” ‘ ’) to straight quotes (" ').
   - Output the result as plain text only.

5. **Structure:**
   - Keep the text as a continuous flow suitable for a single .txt file. 
   - Ensure the chapter heading is included at the top if present.

				[PASTE YOUR TEXT BELOW]
