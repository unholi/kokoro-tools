Section 1

Introduction

Kokoro Audiobook Studio is a locally-hosted web application that converts plain text files into spoken-word audio using the Kokoro TTS (text-to-speech) engine. It is designed for authors, hobbyists, and anyone who wants to produce audiobook-quality narration from their writing — entirely on their own hardware, with no internet connection required after initial setup, and no ongoing API costs.

The studio provides a browser-based interface where you can browse your chapter files, audition voices, blend custom voices, and launch conversion jobs — all from a single tab. Audio is produced as standard WAV files at 24 kHz that can be played, edited, or published with any audio software.

Who this is for: This guide assumes you are comfortable using a Linux terminal for setup. Once installed, the studio itself requires no command-line interaction for day-to-day use.
Section 2

About Kokoro TTS

Kokoro is an open-source, 82-million-parameter text-to-speech model developed by hexgrad and published on Hugging Face. It is notable for producing high-quality, natural-sounding narration from a model small enough to run on a consumer CPU or GPU — no cloud service, no subscription, no data leaving your machine.

The model uses espeak-ng for grapheme-to-phoneme conversion (turning written text into phoneme sequences) and a neural acoustic model to synthesise speech from those phonemes. Voice characteristics are stored as small .pt tensor files that can be loaded, blended mathematically, and passed directly to the pipeline.

How the Studio uses Kokoro

The studio wraps Kokoro's Python API in a lightweight local web server. When you click Launch, the server writes a shell script and opens it in a terminal window — this means the full Kokoro pipeline (which loads PyTorch and model weights) runs in its own process, with live progress visible on screen, without blocking the browser UI.

Voice blending works by loading two .pt tensors and computing a weighted sum — for example, 0.6 × voice_A + 0.4 × voice_B — then passing the resulting tensor directly to the pipeline as a raw embedding rather than a named voice. This bypasses any network lookup entirely.

Section 3

Installing Kokoro

Platform note: These instructions are written for Ubuntu 24.04 / Pop!_OS 24.04. Package names and paths may differ on other distributions. Adjust apt commands for your package manager as needed.

Prerequisites

  1. Python 3.10 or later. Check with python3 --version.
  2. espeak-ng — required by Kokoro's phonemiser:
    sudo apt install espeak-ng
  3. A working pip. If missing: sudo apt install python3-pip

Create a virtual environment

Installing Kokoro into a dedicated virtual environment keeps its dependencies isolated from your system Python. The studio expects the venv to live at ~/Downloads/kokoro/env1/ by default (this path is set in the kokoro-studio launcher script and can be changed).

mkdir -p ~/Downloads/kokoro
cd ~/Downloads/kokoro
python3 -m venv env1
source env1/bin/activate

Install Kokoro and dependencies

# Core Kokoro package
pip install kokoro

# Audio I/O
pip install soundfile

# PyTorch — CPU-only build (smaller download, sufficient for most systems)
pip install torch --index-url https://download.pytorch.org/whl/cpu

# Optional: GPU-accelerated build (much faster on NVIDIA cards)
# pip install torch --index-url https://download.pytorch.org/whl/cu121

First run — download model weights

The first time Kokoro runs it will download the model weights (~330 MB) and any voice .pt files from Hugging Face. This requires an internet connection once. All subsequent runs are fully offline.

cd ~/Downloads/kokoro
source env1/bin/activate
python3 -c "from kokoro import KPipeline; KPipeline(lang_code='a')"
# Wait for download to complete, then Ctrl-C

Pre-cache all voice files

To download all 50+ voice files in one go while online:

python3 - <<'EOF'
from kokoro import KPipeline
import re, os
from pathlib import Path

voices_txt = Path("~/.local/bin/kokoro-tools/voices.txt").expanduser()
voices = []
for line in voices_txt.read_text().splitlines():
    line = line.strip()
    if not line or line.lower().startswith("upload"):
        continue
    name = re.sub(r"\.pt$", "", os.path.basename(line), flags=re.I)
    if re.match(r"^[a-z]{2}_[a-z0-9_]+$", name):
        voices.append(name)

pipeline = KPipeline(lang_code='a')
for v in voices:
    try:
        next(iter(pipeline("test", voice=v, speed=1)))
        print(f"cached: {v}")
    except Exception as e:
        print(f"failed: {v} — {e}")
EOF

Lock to offline mode

Once all voices are cached, prevent any accidental network calls:

echo 'export HF_HUB_OFFLINE=1' >> ~/.bashrc
source ~/.bashrc

Section 5

Using the Studio

Starting the studio

cd ~/your-audiobook-project
kokoro-studio

This activates the Kokoro virtual environment, starts serve.py on port 7891, and opens http://localhost:7891 in your default browser. The terminal window that launched the studio must remain open while you work — closing it stops the server.

The Kokoro Audiobook Studio header is pinned to the top of the page and stays visible as you scroll down through the panels. A ? Help link in the header opens this guide in a new tab at any time.

Voice Audition panel

Click Voice Audition ▶ show to expand. Click any ▶ button next to a voice card to play its sample. Click ■ or click again to stop. Toggle between Native language and Accented English to hear non-English voices either in their native tongue or speaking English with their accent. The Accent and Gender filter rows are independent — gender selection persists as you switch accents. Click Use on any card to set that voice as the active global voice.

If no samples appear, use the ⚙ Native language or ⚙ Accented English buttons to generate them — a terminal window opens and processes all voices automatically. This only needs to be done once; samples are saved in kokoro-tools/voice_samples/ and reused on every future launch.

Note: American (af_ / am_) voices are intentionally excluded from the Accented English sample set — they are native English speakers and produce no accent effect.

Voice Blender panel

Click Voice Blender ▶ show to expand. Select two voices, choose a gender for the output file (cf_voice for female, cm_voice for male), set the mix ratio using the percentage buttons (Voice 1 gets your chosen percentage; Voice 2 gets the remainder automatically), then click ⚗ Blend Voices. Generation takes 20–40 seconds. Once complete, click ▶ to preview and Use for audiobook ▾ to set the blended voice as the active selection.

Blending rule: Only blend voices from the same language family — American with American, British with British, etc. Mixing voice families produces a PyTorch tensor shape error because the embeddings have different dimensions.

Settings

The Settings panel sits between Voice Blender and Sample Audio and applies globally to all conversion operations.

SettingDescription
SpeedNarration speed multiplier. 1.0× is normal; 0.9× is a slightly slower, more measured audiobook pace. Range: 0.5×–2.0×.
Language / Accent pipelineControls the phoneme processing pipeline. Use American English (a) for standard narration. Match to your voice's language family for non-English voices; mismatching causes degraded pronunciation.
Output folderWhere generated WAV files are saved, relative to your working directory. Defaults to ./audiobook_output. Also used by Sample Audio for its output file.

Sample Audio panel

The Sample Audio panel is open by default and can be hidden with the ▾ hide toggle. It lets you generate a short spoken test from any block of text using the currently active voice — useful for checking pronunciation, pacing, or how a specific passage will sound before committing to a full chapter conversion.

  1. Set the active voice using Use in the Voice Audition panel or Use for audiobook ▾ in the Voice Blender. The active voice is displayed read-only in the grey bar at the top of the Sample Audio panel — it cannot be changed from within this panel.
  2. Paste or type any text into the text box.
  3. Click ▶ Generate Audio. The audio is generated inline (no terminal window opens) and saved as sample_audio.wav in your configured output folder.
  4. Click the ▶ play button to hear the result through the floating player. The play button activates only after a successful generation.
Speed and language pipeline settings from the Settings panel are applied to Sample Audio generation as well. Adjust those before generating if needed.

Text Files and converting chapters

The Text Files panel lists every .txt file found in your current working directory, along with a line count and word count for each file (blank lines are excluded from the line count). The table refreshes automatically on load; click ⟳ Refresh if you add files after launching the studio.

  1. Tick the checkboxes next to the chapters to convert. Use Select all or Deselect all for bulk selection.
  2. Assign a voice per file using each row's dropdown, or use the Apply voice to all selected bar at the top to set one voice across all ticked files at once. Custom voices (cf_voice / cm_voice) appear at the top of each dropdown after being created.
  3. Confirm Speed, Language pipeline, and Output folder in Settings.
  4. Click ▶ LAUNCH. A terminal window opens showing chunk-by-chunk progress. WAV files appear in your output folder as each chapter completes.

Session log

Every conversion is appended to kokoro_session.log inside your output folder. Each entry records the timestamp, filename, voice, speed, output duration, and status — useful for replicating a specific result or auditing what was used for each chapter.

Section 6

Troubleshooting

Terminal window opens and closes immediately

Symptom: Clicking Launch opens a terminal that disappears in under a second with no audio produced.

Cause: The terminal is finding a different Python than the one with Kokoro installed — typically /usr/bin/python3 (system Python) instead of the virtualenv Python. The most common error is ModuleNotFoundError: No module named 'numpy' or similar.

Fix: Ensure you launch the studio via the kokoro-studio script, not by running python3 serve.py directly. The launcher activates the venv first, so serve.py inherits the correct Python. Verify by checking the startup banner — it should show the venv path under Venv:

Venv   : /home/yourname/Downloads/kokoro/env1
Python : /home/yourname/Downloads/kokoro/env1/bin/python3

If it shows /usr/bin/python3, edit kokoro-studio and update the VENV_ACTIVATE path to match your actual venv location.

Permission denied when running kokoro-studio

Symptom: bash: /home/user/.local/bin/kokoro-tools/kokoro-studio: Permission denied

Fix: The execute bit is not set. Run:

chmod +x ~/.local/bin/kokoro-tools/kokoro-studio

Cannot reach serve.py / UI shows ⚠ error

The browser UI communicates with serve.py on http://localhost:7891. If the UI shows a connection error, check that:

  • The terminal that launched kokoro-studio is still open
  • Nothing else is using port 7891 (lsof -i :7891)
  • You opened the browser after the server started (the launcher waits 1 second; on slow machines you may need to refresh)

Sample Audio play button does nothing after generating

Symptom: Clicking the ▶ play button in Sample Audio has no effect, or the floating player appears but produces no sound.

Cause 1 — generation failed silently: The play button only enables after a confirmed successful generation. If the status line shows an error, the WAV file was not written. Check the serve.py console for the underlying Python error.

Cause 2 — stale session: The sample_audio.wav path is stored in memory by serve.py. If the server was restarted after generation, the path is lost. Re-generate the sample to restore playback.

Cause 3 — output folder does not exist: If the configured output folder path is invalid or the directory could not be created, the file write fails silently. Check that the output folder field in Settings contains a valid path.

Voice Blender fails with tensor shape error

Symptom: Blend fails with a PyTorch shape mismatch error in the serve.py console.

Cause: You are trying to blend voices from different language families (e.g. an American voice with a Japanese voice). Their embedding tensors have different shapes and cannot be added directly.

Fix: Only blend voices with the same two-letter prefix group — American with American, British with British, etc. Cross-language blending is not supported by Kokoro's architecture.

Custom voice (cm_voice / cf_voice) sounds like one voice only

If the blend sounds identical to one of the source voices, the mix ratio is likely at an extreme (90%/10% or 10%/90%). Try a more balanced ratio such as 60%/40% or 50%/50% for a noticeable blend.

Mispronounced character or place names

Kokoro uses espeak-ng for phoneme conversion, which handles standard English well but often mispronounces invented or unusual proper nouns. The workaround is phonetic substitution directly in the source .txt file — write the name as it sounds rather than how it is spelled. Keep a substitution log so you can apply corrections consistently across all chapters.

Curly quotes producing odd pauses or skipped words

Word processors save "smart" curly quotes (" " ' ') which can confuse the phonemiser. Convert all files to straight quotes before processing:

sed -i "s/\u2018/'/g; s/\u2019/'/g; s/\u201C/\"/g; s/\u201D/\"/g" *.txt

Generated audio sounds like a "tin can" or has interference

Already fixed in current version. Earlier versions of the studio played two Audio objects simultaneously — the interference between them caused phase cancellation (the "tin can" effect). The current version uses a single <audio> element exclusively.

HF_HUB_OFFLINE causes errors after setting it

If you set HF_HUB_OFFLINE=1 but some voice .pt files were not cached before going offline, Kokoro will fail to load those voices. Either temporarily unset the variable and re-run with the missing voices while online, or manually copy the .pt files from another machine into the cache directory:

~/.cache/huggingface/hub/models--hexgrad--Kokoro-82M/snapshots/<hash>/voices/
